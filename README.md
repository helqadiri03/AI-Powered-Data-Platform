# AI-Powered E-Commerce Data Platform

A comprehensive, full-stack data platform designed to bridge the gap between complex data infrastructure and business users. This project combines a robust modern data stack (Airflow, Snowflake, dbt) with advanced AI capabilities (Text-to-SQL, RAG) and enterprise-grade monitoring.

---

## 🏗️ Architecture & Technology Stack

### 1. Data Infrastructure & ELT
*   **PostgreSQL**
    *   *Why:* Used as the robust, transactional relational database for storing raw source data (simulating a production DB) and as the metadata backend for Apache Airflow.
*   **Snowflake**
    *   *Why:* A cloud-native Data Warehouse chosen for its near-infinite scalability and separation of storage and compute. It serves as the analytical engine executing the queries generated by the AI.
*   **dbt (Data Build Tool)**
    *   *Why:* Brings software engineering best practices to data transformation. Used to transform raw ingested data in Snowflake into a clean, optimized Kimball star schema (Facts & Dimensions) ready for analytics.
*   **Apache Airflow**
    *   *Why:* The industry standard for data orchestration. Used to schedule and monitor the end-to-end ELT pipeline: extracting data from Postgres, loading it into Snowflake, and triggering dbt transformations.

### 2. Artificial Intelligence (AI) Layer
*   **Groq (Llama-3.3-70b-versatile)**
    *   *Why:* Provides ultra-low latency LLM inference. Groq's LPU architecture ensures that Text-to-SQL generation and RAG responses happen in milliseconds, providing a seamless user experience.
*   **Pinecone**
    *   *Why:* A fully managed, serverless vector database. Used to store document embeddings for the RAG (Retrieval-Augmented Generation) pipeline, allowing the AI to search and retrieve relevant documentation instantly.
*   **LangChain**
    *   *Why:* A framework for developing applications powered by language models. Used to orchestrate the RAG pipeline, connecting document loaders, text splitters, Pinecone retrievers, and the Groq LLM.
*   **SQLGlot**
    *   *Why:* Used for parsing, validating, and sanitizing the SQL generated by the LLM before executing it against Snowflake, ensuring security and syntax correctness.

### 3. Application Layer
*   **FastAPI (Python)**
    *   *Why:* A modern, fast, async-first web framework for Python. Serves as the backend API bridging the React frontend with the AI models and Snowflake. Chosen for its native Pydantic integration, automatic Swagger documentation, and high performance.
*   **React (Vite + TypeScript)**
    *   *Why:* A powerful frontend library paired with Vite for lightning-fast builds. Used to build the business-facing UI containing the Text-to-SQL chat interface and the AI Documentation Assistant.

### 4. Monitoring & Observability
*   **Prometheus**
    *   *Why:* The standard for time-series metrics collection. Configured to scrape custom operational metrics from the FastAPI application and Airflow (via StatsD).
*   **Grafana**
    *   *Why:* The premier open-source visualization tool. Connected to Prometheus to provide engineers with real-time dashboards tracking API latency, error rates, AI generation times, and system resource usage.

### 5. Infrastructure
*   **Docker & Docker Compose**
    *   *Why:* Used to containerize every component of the stack (Airflow, Postgres, FastAPI, React, Prometheus, Grafana, StatsD). Ensures that the entire complex architecture can be spun up locally with a single command, eliminating "it works on my machine" issues.

## 🚀 How to Set Up and Run

### Prerequisites
1.  **Docker** and **Docker Compose** installed on your machine.
2.  A **Snowflake** account (for data warehousing).
3.  A **Groq** API key (for LLM inference).
4.  A **Pinecone** API key and an index named `ecommerce-docs` (Dimension: 1024, Metric: Cosine).

### Step 1: Environment Configuration
Create a `.env` file in the root directory and populate it with your credentials:

```env
# Database
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_DB=ai_powered_eco_db

# Snowflake
SNOWFLAKE_ACCOUNT=your_account_locator
SNOWFLAKE_USER=your_user
SNOWFLAKE_PASSWORD=your_password
SNOWFLAKE_DATABASE=ECOMMERCE
SNOWFLAKE_SCHEMA=RAW
SNOWFLAKE_WAREHOUSE=COMPUTE_WH
SNOWFLAKE_ROLE=ACCOUNTADMIN

# Groq
GROQ_API_KEY=gsk_...

# Pinecone
PINECONE_API_KEY=pcsk_...
PINECONE_INDEX=ecommerce-docs

# Airflow
FERNET_KEY=YOUR_FERNET_KEY_HERE
AIRFLOW__API_AUTH__JWT_SECRET=YOUR_JWT_SECRET_HERE

# Grafana
GRAFANA_ADMIN_USER=admin
GRAFANA_ADMIN_PASSWORD=admin
```

### Step 2: Build and Start the Infrastructure
Run the following command in the root directory to build all images and start the containers in the background:

```bash
docker compose up -d --build
```

### Step 3: Accessing the Services
Once all containers are healthy, you can access the various parts of the platform:

*   **Business UI (React)**: [http://localhost:5173](http://localhost:5173)
    *   *Use this to interact with the Text-to-SQL agent and the AI Documentation Assistant.*
*   **Backend API Docs (FastAPI/Swagger)**: [http://localhost:8000/docs](http://localhost:8000/docs)
    *   *Use this to view and test the raw API endpoints.*
*   **Data Orchestration (Airflow)**: [http://localhost:8080](http://localhost:8080)
    *   *Login with `airflow` / `airflow`. Use this to trigger the `ecommerce_etl_pipeline` DAG.*
*   **Observability Dashboard (Grafana)**: [http://localhost:3000](http://localhost:3000)
    *   *Login with `admin` / `admin`. Go to Dashboards -> "AI E-Commerce API Dashboard" to view live metrics.*

### Step 4: Data Ingestion (First Time Setup)
1.  Open Airflow at [http://localhost:8080](http://localhost:8080).
2.  Enable the `ecommerce_etl_pipeline` DAG.
3.  Trigger the DAG manually to extract data from Postgres, load it into Snowflake, and run the dbt transformations.
4.  Once completed, the Snowflake `MARTS` schema will be populated and ready for the Text-to-SQL agent.

### Step 5: Document Ingestion (First Time Setup)
To populate the Pinecone vector database with your documentation:
```bash
# Run this from within the FastAPI container or your local environment
python api/rag/ingest.py
```
